Framer Motion

A Beginner's guide

Fais Edathil

Introduction

Framer Motion is an animation library for React js. It is very easy to learn and beginner friendly as compared to vanilla CSS aniations.Using framer motion in our project will speed up our work.

Installation

npm install framer-motion

After Installation, need to import a motion component to our project as follows:-
import {motion} from 'framer-motion'

Animate Attribute

Let's start animating our first component.
First step is very simple, just add "motion." before every tags.
example: if you are animating h1,
instead of <h1> </h1> change to <motion.h1> </motion.h1>
Now we have to assign an animate attribute, and define what kind of animation we want.
animate = {{x: " 100px "}}
this will animate our element 100px away from actual position

feel free to play around other animation properties like opacity,rotateX,background-color, font-size. etc
Once we learn the basics ,Always try to read the official docs for better understanding.

Initial attribute

Now we can add an initial attribute so that we can customize our animation.
Initial attribute as the name indicates , is used to define the the beginning of our animation.
Let's see an example
<motion.h1 initial={{opacity:0}} animate={{opacity:1}} >
Hello World
</motion.h1>
Let's see this in action

Transition Attribute

Transition attribute gives us more control and customisation on our project.
Let's learn some properties that can be used inside transition attribute
delay property:
If we set delay:1
then the animation will begin only after one second.
See the sample code below:
<motion.h1
initial={{opacity:0}} animate={{opacity:1}} transition={{delay:1}} >
Hello World
</motion.h1>
Let's try some transition properties such as duration, type,stiffness etc

Animation On Over

Just add a property WhileHover and there you go!
Let's learn with an example
<motion.button
WhileHover={{scale:1.1}} >
Sample Button
</motion.button>

Variants

We have learnt enough basics, congratulations !!. Let's learn some advanced stuffs now. Imagine we are working on a big project, defining animation properties in every line will be extremely difficult.
Thats exactly were Variants come for our help. In this method , we have to define our animations as an object .
const myVariants = {
start: {x:'-100vw'},
final: {x:0,
transition: { delay:1, stiffness:120 }
} }
myVariants, start, final are my choice of names. You can use any name you wish. <motion.h1
Variants={myVariants} Initial="start" Animate="final" >
Sample Button
</motion.h1>


Side Note: Here variants are assigned as an object, hence inside curly bracket. While Initila and animate properties are assigned as a string
